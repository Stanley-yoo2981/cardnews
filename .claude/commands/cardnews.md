# 오늘의 카드뉴스 초안 생성

CLAUDE.md의 절대 규칙을 먼저 읽어라. 위반 시 즉시 중단한다.

## 순서

1. `calendar.json` 을 읽고 `published: false` 이면서 `date` 가 가장 이른 항목 **하나**를 고른다.
   해당 항목이 없으면 "발행 대기 항목 없음"만 출력하고 종료한다.

2. 그 항목의 `url` 을 WebFetch 로 가져온다. 본문 전체를 확보한다.

3. 본문에서 검토 변호사 이름을 추출한다.
   정규식: `(유영규|김환섭|홍기웅|김선호)\s*변호사`
   **못 찾으면 여기서 중단하고 보고한다.** 추측하지 마라.

4. 본문을 10장으로 분해한다. 각 장은 아래 역할을 가진다.

   | 장 | 역할 | 문구 분량 |
   |---|---|---|
   | 1 | 도입 — 멈추게 하는 한 문장 | 헤드라인 + 인용구 1줄 |
   | 2 | 궁금증 확장 — 왜 지금 중요한가 | 헤드라인 + 2줄 |
   | 3 | 문제 정의 — 사람들이 멈추는 지점 | 헤드라인 + 2줄 |
   | 4 | 관점 전환 — 당연한 가정을 깬다 | 헤드라인 + 2줄 |
   | 5 | 법리 대비 — 막힌 조문 vs 열린 조문 | 대비 블록 2개 |
   | 6 | 핵심 (저장 포인트) + 사건 결과 | 헤드라인 + 결과 스탯 3개 |
   | 7 | 적용 방식 — 실무 자료 3가지 | 넘버 카드 3개 |
   | 8 | 결정적 관점 — 가장 강한 통찰 | 헤드라인 + 2줄 |
   | 9 | 실행 체크리스트 4개 | 짧은 구 4개 |
   | 10 | 상담 안내 | 고정 (수정 금지) |

   6장의 결과 스탯은 **원문에 명시된 법원 판단만** 쓴다. 없으면 스탯 블록을 비운다.

5. `template.html` 을 복사해 `drafts/<date>_<id>/index.html` 로 만들고 문구를 주입한다.
   - `h1`, `.kicker`, `.sub`, `.card p`, `.stats` 만 교체한다.
   - 10장의 상담 블록·면책 문구·광고책임변호사 표기는 **손대지 않는다.**
   - 변호사 사진은 `assets/lawyers/` 의 해당 파일을 base64 로 인라인한다.

6. `node scripts/compliance.mjs drafts/<date>_<id>/index.html` 실행.
   실패하면 원인을 고치고 다시 실행한다. 통과할 때까지 렌더링하지 않는다.

7. `node scripts/render.mjs drafts/<date>_<id>` 실행 → PNG 10장.

8. `caption.txt` 작성 — 인스타 본문 3~4줄 + 상담번호 + 면책 1줄 + 해시태그 8개 이내.

9. `review.md` 작성 — 표 형식.
   각 슬라이드 문구가 **원문의 어느 문장에서 나왔는지** 근거를 남긴다.
   근거를 못 대는 문장은 지워라. 이 표가 변호사 검수의 핵심이다.

10. 마지막에 아래 한 줄만 출력한다.

    `초안 완료: drafts/<date>_<id> · 검토 변호사: OOO · 검수 후 수동 발행 필요`

## 하지 않는 것

- 게시하지 않는다.
- `calendar.json` 의 `published` 를 바꾸지 않는다. 사람이 발행 후 직접 바꾼다.
